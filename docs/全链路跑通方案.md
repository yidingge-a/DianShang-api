# 全链路跑通方案

## 架构

```
浏览器 → Vite(8080) 或 Nginx(80)
           ├─ /api/v1/*  → FastAPI(8000)
           └─ /uploads/*   → 静态文件 / 对象存储
FastAPI → SQLite(开发) / PostgreSQL(生产)
        → Redis + Celery Worker（上架、视频异步）
        → LLM / 第三方电商数据 API（可配置）
```

## 本地开发（推荐）

### 1. 后端（WSL）

```bash
cd DianShang_project
cp .env.example .env
bash scripts/sync_env_from_deploy.sh   # 同步 OPENAI_API_KEY
bash scripts/start_backend.sh          # DEBUG=true，:8000
```

默认账号：`dev@local.test` / `dev123456`

### 2. 前端（Windows）

```bash
cd D:\KIMI_project\电商项目\nocode\nocode
npm install
npm run dev   # :8080，代理 /api → 127.0.0.1:8000
```

`.env.development` 已设置 `VITE_API_URL=/api/v1`、`VITE_DISABLE_NOCODE=1`。

### 3. 验证

```bash
# 契约测试（61 项）
python3 scripts/test_api_v2.py

# 真实联调（后端 + 前端 + Playwright）
bash scripts/run_e2e_live.sh
```

## 生产部署

见 [deploy/PRODUCTION.md](../deploy/PRODUCTION.md)：

- `DEBUG=false`，强 `SECRET_KEY`
- PostgreSQL + `alembic upgrade head`
- `CELERY_ENABLED=true` + Redis Worker
- 前端 `npm run build` → Nginx 静态托管
- `docker compose up` 一键启动（api / worker / postgres / redis / nginx）

## 鉴权说明

| 环境 | 行为 |
|------|------|
| `DEBUG=true` | 测试脚本可登录 dev 用户；前端需登录后访问业务页 |
| `DEBUG=false` | 所有业务 API 必须 Bearer Token；无 dev 绕过 |

公开接口：`GET /health`、`POST /api/v1/auth/register|login|refresh`

## 能力矩阵

| 模块 | 实现 | 备注 |
|------|------|------|
| 用户认证 | SQLite/PG + bcrypt + JWT | 真实落库 |
| 比价/市场/运营 | LLM → 电商数据 API → 结构化兜底 | 配 `ECOMMERCE_DATA_API_*` 可接真数据 |
| 智能美工 | Pillow / 可选视觉模型 | 详情页 HTML 落盘 |
| 视频 | 第三方 API → ffmpeg 合成 | 需 ffmpeg |
| 上架 | 发布网关 → 智能 listing HTML | 配 `PUBLISH_GATEWAY_*` |
| 异步任务 | Celery（可选） | 上架、长视频 |

## 常见问题

**登录页空白**：确认 `VITE_DISABLE_NOCODE=1`，重启 `npm run dev`。

**401 未登录**：前端需先登录；API 测试需带 `Authorization: Bearer <token>`。

**LLM 超时**：检查 `OPENAI_API_KEY` 与 `OPENAI_BASE_URL`；未配置时走本地兜底数据。
